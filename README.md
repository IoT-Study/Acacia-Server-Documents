# Acacia サーバー
Acaciaサーバー班のドキュメント管理リポジトリです。  
Issue、仕様、開発に必要な情報を管理します。

## フォルダ構成
```
Documents/			#参考資料フォルダ
Images/				#画像フォルダ
README.md			#このファイル
Specifications/		#仕様書フォルダ
```

## Github運用ルール
### Issue
 - Acaciaプロジェクト参加者なら誰でも作ってOKです。
 - 仕様の追加・変更がある場合は、必ず作ってください。

### ブランチ運用
3つのブランチで運用します。

 - master branch
   - メインのブランチです。決定済みの仕様になります。
 - feature branch
   - 作業用ブランチです。Issue番号で作成してください。  
   - (e.x. feature/gh-123) 
 - fix branch
   - Issueなしで修正したいときに作成してください。プルリクに対する提案などを想定しています。  
   - (e.x. fix/gh-123-{修正内容}) 

### Pull Request
 - 仕様の追加・変更（__Specifications__以下に変更を入れる場合）は、__master__ に __Pull Request__ してください。
 - 仕様の追加・変更以外はプルリク不要です。masterに直接pullしてください。
 - サーバー以外の班からのプルリクも歓迎します。
 
### レビュー
 - 仕様把握のため、できるだけサーバ班全員でレビューするよう心がけてください。
  
### マージ
 - マージは原則リーダーがします。
